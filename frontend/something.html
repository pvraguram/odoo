19. Print All Prime Numbers Between Two Intervals
class PrimeInterval {
    static boolean isPrime(int num) {
        if(num < 2) return false;
        for(int i = 2; i <= Math.sqrt(num); i++)
            if(num % i == 0) return false;
        return true;
    }
    public static void main(String[] args) {
        int start = 10, end = 50;
        for(int i = start; i <= end; i++)
            if(isPrime(i))
                System.out.print(i + " ");
    }
}

BOXING, UNBOXING, TYPE CASTING, AND WRAPPER CLASSES
class BoxingUnboxingCasting {
    public static void main(String[] args) {
        int primitive = 10;
        Integer boxed = primitive;  // Auto-boxing
        int unboxed = boxed;  // Auto-unboxing
        
        double widened = primitive;  // Widening
        int narrowed = (int) widened;  // Narrowing
        
        Integer wrapperInt = Integer.valueOf(25);
        Double wrapperDouble = Double.valueOf(12.5);
        Character wrapperChar = Character.valueOf('A');
        
        System.out.println("Boxed Integer = " + boxed);
        System.out.println("Unboxed Integer = " + unboxed);
        System.out.println("Widening (int to double) = " + widened);
        System.out.println("Narrowing (double to int) = " + narrowed);
        System.out.println("Wrapper Integer = " + wrapperInt);
        System.out.println("Wrapper Double = " + wrapperDouble);
        System.out.println("Wrapper Character = " + wrapperChar);
    }
}

FREQUENTLY ASKED QUESTIONS
1. What is the reason Java doesn't support multiple inheritance for classes?
Java avoids multiple inheritance for classes to prevent the diamond problem and ambiguity issues. When a class inherits from multiple parent classes that have methods with the same signature, it becomes unclear which method should be invoked, leading to complexity and potential conflicts in the codebase.
2. How does Java handle parameter passing for objects?
Java uses pass-by-value for all parameters, including object references. When you pass an object to a method, a copy of the reference is passed. This means modifications to the object's internal state will affect the original object since both references point to the same memory location. However, reassigning the reference itself within the method won't change the original reference in the calling scope.
3. What's the rationale behind Java's exclusion of explicit pointers?
Java eliminates explicit pointer usage to enhance memory safety and security. By abstracting memory management through object references and automatic garbage collection, Java prevents common pointer-related errors such as memory leaks, dangling pointers, and unauthorized memory access. This design promotes a more secure and robust programming environment.
4. How are arrays passed to methods in Java?
Java passes arrays using pass-by-value semantics, where the value is the reference to the array object. A copy of the array reference is passed to the method, allowing modifications to array elements to affect the original array since both references point to the same memory location. However, if you reassign the reference to a new array within the method, the original array in the calling context remains unchanged.
5. What happens when objects are passed to methods in Java?
Java employs pass-by-value for objects, where the value passed is the object's reference. A copy of this reference is sent to the method, meaning any changes to the object's properties through the copied reference will be reflected in the original object because both references point to the same instance in memory. However, reassigning the reference variable inside the method doesn't affect the original reference outside the method.
